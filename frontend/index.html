What is this project?

A bouncy physics simulation running on the LMAX Disruptor — the fastest inter-thread messaging library in the JVM — compiled to WebAssembly and running in your browser. The right panel shows a live view of what's happening inside the concurrency layer: lock waits, queue depths, spin counts, thread states. Press CHAOS MODE to stress-test it.

⬡ How ThreadScope Works
① Your Java Application
Any JVM-based service — Spring Boot, Disruptor trading engine, microservice, batch processor. ThreadScope attaches to its JVM with zero code changes.
REAL JVM
▼
② ThreadMXBean Collection
java.lang.management.ThreadMXBean.dumpAllThreads() captures every thread's state (RUNNABLE, BLOCKED, WAITING), CPU%, lock names, and stack traces 10× per second.
JDK BUILT-IN
▼
③ LMAX Disruptor Engine
A real RingBuffer[1024] with 3 producer and 3 consumer threads runs inside the agent. This generates genuine contention, real latency measurements, and authentic backpressure — no simulation.
LIVE DATA
▼
④ AI Root Cause Engine
5 diagnostic rules evaluated every 100ms using 30-tick sliding windows. Detects Deadlock, Lock Contention, Consumer Backpressure, CPU Saturation, Memory Pressure. Confidence scores derived from metric signal strength — not random. Rule-based expert system, not an LLM.
AI ENGINE
▼
⑤ WebSocket Stream
Spring Boot broadcasts a JSON snapshot — threads, disruptor metrics, JVM stats, AI diagnosis — to every connected browser at 100ms intervals via the configured WebSocket endpoint.
10 FPS
▼
⑥ Physics Dashboard
Real-time HTML5 Canvas physics simulation maps thread states to particles — RUNNABLE (cyan), BLOCKED (red), WAITING (gold). Every particle represents a real JVM thread. Chaos mode spawns real contention threads and instantly reshapes the visualization.
CANVAS 60FPS
Is This Actually AI?
Yes — it is a rule-based expert system, the same architecture used in production APM tools (Datadog, New Relic, Dynatrace). It applies domain-expert rules to time-series signals with weighted confidence scoring and trend analysis. The difference from a static dashboard: it correlates multiple metrics simultaneously, tracks trends across 3-second windows, and generates a specific root cause + actionable recommendation — not just an alert.
What Problem Does It Solve?
Backend engineers see "high latency" in production but spend hours guessing whether it is lock contention, CPU saturation, GC pressure, or queue backlog. ThreadScope makes the invisible visible in real time and tells you exactly what is wrong and what to fix — in seconds, not hours.
